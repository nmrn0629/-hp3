# 管理者向けトピックス更新マニュアル（複数人運用編）

このドキュメントでは、複数の管理者がトピックスを更新するための環境設定と、フォルダ構造について解説します。

## 1. ディレクトリ構造の解説
サイトのデータが入っているメインフォルダ（`-hp3`）の中に、**`js/`** というフォルダがあります。

- **`js/` とは？**: 「JavaScript（ジャバスクリプト）」の略で、ホームページに「動き」や「データ」を与えるプログラム・ファイルをまとめて置く場所です。
- **`js/topics-data.js`**: トピックスの最新データが書き込まれる「データ室」のようなファイルです。ここを書き換えることで、全ページのトピックスが更新されます。

## 2. 複数管理者で運用するための環境構築
他のパソコン（別の担当者のPCなど）からも更新作業を行えるようにするには、以下の「一回切り」の設定が必要です。

### 必要なもの
1.  **Gitのインストール**: 更新した内容をGitHubへ送信するためのツールです。
2.  **GitHubのアクセス権限**: GitHub上でプロジェクトに協力者（Collaborator）として追加されている必要があります。
3.  **プロジェクトの「クローン（コピー）」**: GitHubにある最新のデータを、自分のPCにダウンロードします。

### 他の管理者の設定手順
1.  **Gitをインストールする**:
    - [Git公式サイト](https://git-scm.com/)からダウンロード・インストールしてください。
2.  **GitHubにログインする**:
    - ブラウザでGitHubにログインし、プロジェクトの「Settings」→「Collaborators」から、他の管理者のアカウントを招待してください。
3.  **データをPCに持ってくる**:
    - `git clone https://github.com/nmrn0629/-hp3.git` コマンド等で、自分のPCに作業フォルダを作ります。
4.  **自動送信スクリプトを起動する**:
    - そのPCのフォルダ内にある `auto-push-topics.ps1` を右クリックして「PowerShellで実行」します。これで、あなたのPCと同様に「ファイルを置くだけ」で更新できるようになります。

---

## トピックス更新の鉄則
複数の管理者がいる場合、**「作業を始める前に、必ず最新の状態にする（プルする）」**ことが重要です。

- 黒い画面（PowerShell）を開いた際にエラーが出る場合は、誰か他の人が更新したデータがGitHubに上がっている可能性があります。
- その場合は、一度 `git pull` という操作を行ってから作業を再開してください。
